The archive feature bundles all collected data into a single compressed archive file. This is useful for creating portable snapshots of your infrastructure data.

When enabled, all step outputs are written to the archive, and the archive is then written to the configured sink. This means you can combine archiving with any sink (filesystem, S3, etc.).

## Configuration

<PropertyReference
  schema={archive}
  schemas={{
    "archive-tar": archiveTar,
  }}
/>

### tar

The only supported archive format today. The `compression` attribute controls how the tarball is compressed.

| Compression | Extension | Description |
|-------------|-----------|-------------|
| `gzip` | `.tar.gz` | Good compression ratio, widely supported (default) |
| `zstd` | `.tar.zst` | Better compression ratio and speed |
| `none` | `.tar` | No compression, fastest |

## Examples

### Basic archive to filesystem

```hcl
output {
  archive "tar" {
    compression = "gzip"
  }
  sink "filesystem" {
    path = "./output"
  }
}
```

Creates `./output/<job-name>.tar.gz` containing all step outputs.

### Archive with zstd compression to S3

```hcl
output {
  archive "tar" {
    compression = "zstd"
  }
  sink "s3" {
    bucket = "my-backups"
    region = "us-east-1"
    prefix = "infracollect/"
  }
}
```

### Complete example

```hcl
job {
  name = "infrastructure-snapshot"
}

collector "terraform" "aws" {
  provider = "hashicorp/aws"
  version  = "5.0.0"
  region   = "us-east-1"
}

step "terraform_datasource" "vpcs" {
  collector = collector.terraform.aws
  datasource "aws_vpcs" {}
}

step "terraform_datasource" "instances" {
  collector = collector.terraform.aws
  datasource "aws_instances" {}
}

output {
  encoding "json" {
    indent = "  "
  }
  archive "tar" {
    compression = "gzip"
  }
  sink "s3" {
    bucket = "infrastructure-snapshots"
    region = "us-east-1"
    prefix = "daily/"
  }
}
```
