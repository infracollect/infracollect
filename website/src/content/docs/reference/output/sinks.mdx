Sinks define where collected data is written. You can write to the local filesystem, S3-compatible object storage, or stdout/stderr.

## Configuration

  schemas={{

## Filesystem

Write output files to a directory on the local filesystem.

### Configuration

<PropertyReference schema={sinkFilesystem} />

### Example

```hcl
output {
  sink "filesystem" {
    path = "./output"
  }
}
```

## S3

Write output to S3-compatible object storage. Supports AWS S3, Cloudflare R2, MinIO, and other S3-compatible services.

### Configuration

<PropertyReference
  schema={sinkS3}
  schemas={{
    "sink-s3-credentials": sinkS3Credentials,
  }}
/>

### Examples

#### AWS S3

```hcl
output {
  sink "s3" {
    bucket = "my-bucket"
    region = "us-east-1"
    prefix = "infracollect/"
  }
}
```

Uses the default AWS SDK credential chain (environment variables, shared credentials file, IAM role, etc.).

#### MinIO

```hcl
output {
  sink "s3" {
    bucket           = "my-bucket"
    endpoint         = "http://localhost:9000"
    force_path_style = true
    credentials {
      access_key_id     = "minioadmin"
      secret_access_key = "minioadmin"
    }
    prefix = "data/"
  }
}
```

#### Cloudflare R2

```hcl
output {
  sink "s3" {
    bucket   = "my-bucket"
    endpoint = "https://ACCOUNT_ID.r2.cloudflarestorage.com"
    credentials {
      access_key_id     = env.R2_ACCESS_KEY
      secret_access_key = env.R2_SECRET_KEY
    }
    prefix = "data/"
  }
}
```

## Stdout

Write output to standard output. Useful for piping to other tools or debugging.

```hcl
output {
  sink "stdout" {}
}
```
